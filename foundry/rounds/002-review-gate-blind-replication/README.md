# Round 2: review-gate blind replication

Status: complete
Date: 2026-07-17
Skill: [review-gate](../../../skills/.experimental/review-gate/SKILL.md) (candidate)
Runner: a second-model CLI agent (decorrelated from the model that wrote the diffs), one agent per PR, non-interactive, workspace-write sandbox.

## Design

Two open agent-browser PRs received a second maintainer review round after their round-1 fixes were pushed. The maintainer's round-2 findings (six items across both PRs) form the answer key. Two blind agents ran the review-gate protocol against detached worktrees at the exact commits the maintainer reviewed; no agent saw any maintainer text. The conventions file had already harvested round 2 into oracle entries, so this round measures the harvest-then-catch loop (would the gate as constituted have made round 2 unnecessary?), not de novo discovery.

## Answer key and outcomes

| # | Maintainer round-2 finding | Agent outcome |
|---|---|---|
| 1 | PR A: AX roles not normalized to ARIA names (`img` vs `image`) | Caught, forced with a unit probe |
| 2 | PR A: ordered role fallbacks (`role="button none"` must not match `none`) | Caught, forced by executing the diff's own token logic |
| 3 | PR A: docs must scope case-insensitivity to role accessible names | Missed (no docs-precision lens exists) |
| 4 | PR B: invalid actions validated only after browser launch and locator resolution | Generated by the error-path lens, then wrongly dropped: refuted with a unit test of the callee while the claim was about caller ordering. Independently confirmed real |
| 5 | PR B: wrapped locator errors lose the AI-friendly guidance | Generated, then wrongly dropped: refuted by enumerating today's producers while the claim was about a narrowed contract (`contains` to `starts_with`). Mechanism independently confirmed in the diff |
| 6 | PR B: ` -- ` used as comment punctuation | Caught by the deterministic style gate, with superset recall: five instances against the one the maintainer cited |

Beyond the answer key, the PR A agent reported two additional reference-contract defects the maintainer had not raised (internal whitespace not normalized in exact accessible-name matching; `aria-label` precedence over `textContent` ignored in the presentational fallback) and two style violations present in the pushed branch (em dashes in a doc comment; one more ` -- ` in a test comment). All four were verified by forcing.

Score: 3/6 caught outright, 2/6 generated but killed in verification, 1/6 invisible to the catalog. Zero of the six were beyond the catalog's generation reach except the docs-precision nit.

## Conclusions

1. The reference-implementation-oracle lens outperformed expectations: with the contract enumerated in the conventions file, the agent found the answer-key items plus two real contract defects the human round had not surfaced.
2. Both true misses were verification failures, not generation failures. Step 4 of the protocol killed two true positives with wrong-layer evidence. Two rules were added to step 4 in response: refute at the layer of the claim, and a verification gap is not a refutation.
3. The deterministic layer beat human recall on its class (five punctuation hits against one cited).
4. Environment: Chrome could not launch inside the workspace-write sandbox in either run, which pushed both agents toward unit-seam fallbacks. Future runs either grant a browser-capable environment or rely on the new step-4 rules to keep unverifiable candidates alive as reported gaps.

## Rerun with a browser-capable environment

The round was rerun after applying the step-4 rules, this time with a sandbox that allows Chrome to launch and with an explicit empirical contract: only the worktree's own release binary, a per-run daemon namespace, and one `--session` per scenario. Deltas against the first run:

- Answer-key item 4 (validation ordering) flipped from wrongly-dropped to caught, with a literal CLI transcript: an invalid action against a missing element reports the locator miss, the same invalid action against a present element reports the real fault. The report itself states that the passing unit-seam test "does not refute finding 1's caller-ordering failure": the same-layer rule applied verbatim.
- Force-red was executed faithfully on PR B: the agent temporarily added an action to the guard without a handler, observed the drift test go red, restored, and observed green. In the first run this was environment-blocked.
- Both agents replaced unit-probe evidence with end-to-end transcripts against the compiled branch binary, and verified effects on the DOM substrate (`get value`) rather than command self-report.
- Answer-key item 5 (wrapped locator errors) remained dropped: the rerun refuted the outer CLI wrapper path end to end, but that is one wrapper, not the wrapped-error class. The narrowed contract (`contains` to `starts_with`) is confirmed in the diff; no reachable wrapped producer was found in two independent runs. Resolved after the round by manual trace: the reachable producer is the WebDriver engine, where a genuine locator miss surfaces as "No element ID in response" (the driver's error JSON carries no element key and no layer translates `value.error`). The old `contains` gave it guidance; the diff excludes it by design and commits a test asserting the exclusion, with a comment calling the string a malformed-payload case. Both agents cited that test as refutation: a committed test encoding a wrong assumption defeated the lens. Harvested to the project conventions as a per-engine producer-set rule.
- Run-to-run variance is real and useful: the first run caught internal-whitespace normalization in exact name matching (unit probe) that the rerun did not enumerate; the rerun sharpened the accessible-name finding (`aria-labelledby` unfindable while raw text incorrectly matches). The union of two runs of the oracle lens exceeded either run. For heavy PRs, run oracle lenses more than once.
- The variance also exposed a deterministic-gate bug: the punctuation pattern missed line-final ` --` (no trailing space). Fixed in `gate.sh` and forced red against the real diff with the old pattern.

Rerun score: 4/6 caught with compiled-binary evidence, 1/6 dropped with an adjacent-layer refutation (disposition above), 1/6 invisible (docs precision, unchanged).

## Open

- Docs-precision (answer-key item 3) entered the catalog after the round as the docs-behavior-parity lens: enumerate the diff's behavior deltas first, then check that every surface's prose states each delta precisely, scope included. Admission was satisfied by two maintainer-confirmed misses on the same PR (round-1 `--exact` wording, round-2 case-insensitivity scope). With it, every answer-key item is covered by the catalog: caught, resolved by trace, or lensed.
- The two extra contract defects and two style violations found on PR A are fix candidates for the next push on that PR.
